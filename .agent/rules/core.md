# Core Standards

> Always apply these standards.

## General

- Prefer small, focused changes over large refactors
- Preserve existing patterns unless explicitly asked to change them
- Reference `AGENTS.md` for project-specific setup and conventions

## Before Making Edits

- Understand the scope before editing
- For multi-step work, consider creating a plan in `.agent/plans/`
- Check for existing tests and update or add as needed

## Error Handling

- Handle errors explicitly; avoid silent failures
- Log errors with useful context for debugging

## Documentation

- Update relevant docs when changing behavior
- Prefer inline comments for non-obvious logic
