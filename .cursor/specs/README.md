# Specs

Product specs and requirements live here. PM mode writes specs before implementation.

## Workflow

1. **PM** — Write spec in `.cursor/specs/` (use `_template.md`)
2. **Align** — Confirm scope with stakeholder
3. **Implementation** — Engineer creates plan in `.cursor/plans/` and executes

## Naming

- `feature-*.md` — New features
- `improvement-*.md` — Enhancements
- `fix-*.md` — Bug fix specs (when non-trivial)
